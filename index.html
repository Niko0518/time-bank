版本 v9.17.8 (2026-07-06)

  • 修复：自动检测补录时余额出现暂时性双倍计入
  • 🐛 问题：当自动检测补录（漏记录）或自动修正（多记录）触发一笔交易时，本机的余额显示会出现"双倍计入"——例如漏记 30 分钟应扣 36 分钟（含惩罚），实际本机会扣 72 分钟。屏幕时间自动结算也有同样问题。下次云端同步拉回权威余额后会自动恢复正常，所以表现是"暂时性"。
  • 🔍 根因：addTransaction() 自 v7.36.5 起内置了"按 transaction.type 增量更新 currentBalance"的乐观 UI 逻辑；而 createAutoMakeup、createAutoCorrection、屏幕时间自动结算三处仍在调用 addTransaction() 之前手动 currentBalance +=/-= amount。两次方向一致的余额更新叠加 = 双倍计入。云端 tb_profile.cachedBalance 因走 callMutation → tbMutation 原子更新，没被污染。
  • ✅ 修复：删除三处调用方的手动 currentBalance 更新（app-systems.js 中的 createAutoMakeup / createAutoCorrection / 屏幕时间自动结算），余额更新统一由 addTransaction() 负责。屏幕时间循环里仅保留 totalChange 的累加（用于启动报告汇总展示）。
  • 🛡 幂等性：本修复后 addTransaction() 失败 onRollback 的余额补偿路径天然对齐——以前手动余额更新在失败时不会被回滚，现在完全消除此隐患。
  • 🎯 影响范围：仅影响"自动检测补录/修正"和"屏幕时间自动结算"两类系统任务的余额展示时机；用户手动 startTask / stopTask 等常规路径行为完全不变。

版本 v9.17.7 (2026-07-03)

  • 修复：新设备登录后睡眠/小睡参数不显示云端配置
  • 🐛 背景：用户反馈在 A 设备修改了夜间睡眠计划和小睡参数后，在 B 设备首次登录会看到默认数字（如 plannedBedtime=23:00、napReward=15），不是 A 设备已配置的版本。即使云端 deviceSleepSettings[deviceId] 或 sleepSettingsShared 字段有正确数据，新设备仍显示默认值。
  • 🛠 根因：initSleepSettings 的全新安装保护分支（localUpdated===0 && cloudFormat==='deviceSpecific'）会"保持代码默认值"——这是 v7.33.8 为防止旧格式默认值覆盖新默认值加的保护，但 v9.8.0 之后 sleepSettingsShared 已成为 per-user 跨设备权威，新设备理应继承而不该看默认值。另外 Watch 监听路径上 deviceSleepSettings 回退分支传了 force=false，新设备场景下 localUpdated=0 会被 v7.33.8 保护再次跳过，导致 Watch 兜底也失效。
  • ✨ 修复：① initSleepSettings 的 per-device 格式分支改为「直接采用云端值 + 立即触发 deviceSleepSettings → sleepSettingsShared 迁移」，与 shared 分支同等对待；② Watch 路径的 deviceSleepSettings 回退 force=false → force=true，保证新设备/重装设备也能从云端 per-device 拉回正确配置；③ saveSleepSettings 的 cloudSettings 补全 napMinDurationMinutes 字段（小睡判定阈值，默认 240 分钟），之前该字段从未上云，跨设备同步后会被默认 240 覆盖。
  • 🔍 影响范围：仅 app-sleep.js（initSleepSettings / saveSleepSettings）和 app-1.js（Watch 监听回调），不影响数据层和计费逻辑。新设备首次登录会立即看到 A 设备已配置参数，无需任何额外操作。

时间流图说明

时间流图以日程表的形式展示每日的时间活动轨迹。

• 同时进行的任务会并列显示（最多3列）
• 同一任务暂停后继续会用虚线连接
• 蓝色三角指示当前时间位置
• 向上滚动自动加载更早日期
✨ 任务排序功能

📂 分类栏排序：点击标题旁的 ⇅ 按钮，在弹出的抽屉中拖动调整分类显示顺序。默认云端统一排序，开启"本设备独立排序"可使用本机独立顺序。

🃏 任务卡片排序：长按任务卡片可自由拖动，松手即可完成排序。拖动采用丝滑的物理动画，其他卡片会自动流畅地让出位置。

活动日历是什么？

活动日历以颜色强度呈现每日净时间，直观看到一段时间内的努力密度。

颜色含义
- 净增加：浅绿 0-1小时；中绿 1-3小时；深绿 3小时以上
- 净减少：浅红 0-1小时；中红 1-3小时；深红 3小时以上
交互方式
- 长按显示详情弹窗，可滑动切换日期
- 长按 3 秒后自动进入日详情页
时间概览是什么？

时间概览汇总你在所选周期内的时间表现。

交互方式
- 点击饼图扇区可直接进入分类/任务详情
- 长按显示详情弹窗，可拖动切换其他扇区
- 长按 3 秒自动进入详情页

构成分析是什么？

构成分析用饼图展示你在所选周期内的时间分布。

交互方式
- 切换分类/任务视图，查看不同维度的耗时占比
- 点击扇区可直接进入分类/任务详情
- 长按显示详情弹窗，可拖动切换其他扇区
- 长按 3 秒自动进入详情页
详细数据是什么？

详细数据按表格列出分类/任务的时长与次数，支持筛选、排序和逐步展开更多行。

- 点击表头可按总时间、次数、平均时间排序（再次点击切换升/降序）。
走势分析是什么？

走势分析展示近 7/30 天的每日获得/消费走势，按分类或任务堆叠，直观看到哪些项目在拉升或拖累净时间。

交互方式
- 点击有数据的柱子可直接进入当日详情
- 长按柱子显示提示，可拖动切换日期
- 长按 3 秒自动进入详情页
- 快速左右滑动可切换获得/消费图表
自动检测补录

开启后，系统在每日首次启动时按设备读取关联应用时长，并与您的记录对比后补录/修正。

🤖 漏记补录：实际使用 > 已记录 → 自动补录差额（有一定惩罚）
🔧 多记修正：已记录 > 实际使用 → 自动扣减差额（有一定惩罚）
🔗 多设备：各设备记录自动汇总，避免互相覆盖
🚀 首次启用：不追溯历史，仅从启用当日/次日开始检测

提示：如果自动检测补录失败，可在任务菜单点“🤖 自动检测”触发。

📱 屏幕时间管理

屏幕时间管理帮助您控制手机使用，将节省的时间转化为奖励。

🎯 每日限额：设定每日屏幕使用时间上限
🟢 节省奖励：实际使用 < 限额 → 差额作为奖励存入余额
🔴 超出扣除：实际使用 > 限额 → 超出部分从余额扣除
⚡ 自动结算：每日首次启动时自动结算前一天
🔗 多设备：按设备独立结算，云端自动汇总
📊 分类归属：可在设置中选择奖励/扣除归属的分类

提示：点击或滑动卡片顶部可展开/收起，点击展开后的卡片可进入屏幕时间详情页。

🪟 悬浮窗计时器

悬浮窗计时器让您在使用其他应用时也能随时查看任务进度，点击悬浮窗可快速切换。

⏱️ 计时模式：普通/消费任务显示正计时，达标任务显示倒计时
📱 智能跳转：点击悬浮窗跳转到关联应用或回到时间银行
📚 多任务堆叠：存在多个悬浮窗时，自动堆叠，长按可展开/再次堆叠

提示：设置中为全局开关，任务编辑页可单独设置每个任务的悬浮窗。
