版本 v4.0.0 (2025-10-22)

  • [重构] 移除了底部标签导航栏 (`.bottom-tabs`)。
  • [功能] 新增全屏左右滑动（Swipe）手势来切换“获得”、“消费”、“报告”和“设置”四个主页面。
  • [功能] 滑动切换逻辑会智能避开“报告”页内的图表和表格的横向滚动区域，防止手势冲突。
  • [优化] 强制任务卡片中的任务名称 (`.task-name`) 始终在单行显示，超出部分将显示省略号 (`...`)。

版本 v3.13.0 (2025-10-22)

  • [功能] 调整了消费逻辑：当时间余额为负时，仍可进行“即时兑换”和“持续消耗”类任务。
  • [功能] 新增惩罚机制：当余额为负时执行上述消费任务，其实际消耗时间将按 1.1 倍计算。

版本 v3.12.6 (2025-10-20)

  • [优化] 为所有任务卡片添加了 `2px` 的浅灰色内嵌边框 (`var(--border-color)`)，以解决“习惯任务”的彩色边框导致的视觉大小不一致问题。
  • [优化] 将所有卡片边框（包括习惯任务）的宽度从 `4px` 统一减少到 `2px`，使界面更精致。

版本 v3.12.5 (2025-10-20)

  • [优化] 优化了“习惯任务”卡片的视觉样式，将原有的左侧彩色竖线改为环绕整个卡片的内嵌彩色边框，同时确保不影响卡片布局或交互（通过 `box-shadow: inset` 和 `pointer-events: none` 实现）。

版本 v3.12.4 (2025-10-20)

  • [修复] 修复了 v3.12.3 引入的一个严重布局问题：对仪表盘过滤器（“7天内”等按钮）的右对齐修改错误地影响了“详细数据”和“趋势演变”选项卡的过滤器布局。现已通过高特异性 CSS 规则修复此问题，确保修改只应用在“时间洞察仪表盘”。
  • [修复] 修复了核心 `formatTime()` 函数的逻辑，使其在小时为0、分钟和秒数均大于0时（如 "15分30秒"），能正确显示秒数。此修复影响“时间余额卡片”和“任务计时器”。
  • [优化] 调整了“时间流图”中条形图的间距，将底部空白减少了50% (16px -> 8px)，并将条间距减少了 33% (24px -> 16px)。
  • [优化] 调整了“时间洞察仪表盘”顶部的过滤器布局，使“7天内/30天内/全部”时段按钮组**真正**靠右对齐。
  • [优化] 将“设置”页面中各部分标题（如“外观设置”）与其下方第一项内容之间的间距缩短了50% (24px -> 12px)。
  • [优化] 将“详细数据”表格中的标题 "获得/消费" 修改为 "总时间"，"平均时间" 修改为 "平均"。
