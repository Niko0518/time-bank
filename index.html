版本 v7.40.0 (2026-05-02) ⚡ pendingRegistry 同步架构升级：时间戳结构 + 过期清理

  • [Refactor] 🏆 pendingRegistry 结构升级：`Map` → `Map`，addedAt 时间戳用于超时自动清理，从概率性判断进化为确定性状态判断。
  • [Feature] ⏱️ 自动过期清理：新增 `cleanExpiredPending(timeoutMs)` 函数，默认 5 分钟超时自动清理孤儿 pending 条目，防止内存泄漏。
  • [Refactor] 🗑️ 移除 recentLocalTransactions：完全移除 v7.30.0 引入的 `recentLocalTransactions` Map 和 `RECENT_TX_EXPIRY_MS` 常量，简化代码。
  • [Feature] 📡 deleteTransaction 集成：`DAL.deleteTransaction()` 在云端删除前调用 `addPending(tx)`，Watch remove 回声时自动清理，与 add/update 保持一致。
  • [Fix] 🛡️ Watch update 分支合并：Watch update 处理增加 `isPending(txId)` 检查，收到本机 update 回声时正确忽略，防止远程更新覆盖本地状态。
  • [Fix] 🔧 缩进修复：修复 Watch update 分支 else-if 块内代码缩进错误。

时间流图说明

时间流图以日程表的形式展示每日的时间活动轨迹。

• 同时进行的任务会并列显示（最多3列）
• 同一任务暂停后继续会用虚线连接
• 蓝色三角指示当前时间位置
• 向上滚动自动加载更早日期
✨ 任务排序功能

📂 分类栏排序：点击标题旁的 ⇅ 按钮，长按拖动调整分类显示顺序。

🃏 任务卡片排序：长按任务卡片可自由拖动，松手即可完成排序。拖动采用丝滑的物理动画，其他卡片会自动流畅地让出位置。

活动日历是什么？

活动日历以颜色强度呈现每日净时间，直观看到一段时间内的努力密度。

颜色含义
- 净增加：浅绿 0-1小时；中绿 1-3小时；深绿 3小时以上
- 净减少：浅红 0-1小时；中红 1-3小时；深红 3小时以上
交互方式
- 长按显示详情弹窗，可滑动切换日期
- 长按 3 秒后自动进入日详情页
时间仪表盘是什么？

时间仪表盘汇总你在所选周期内的时间表现。

交互方式
- 点击饼图扇区可直接进入分类/任务详情
- 长按显示详情弹窗，可拖动切换其他扇区
- 长按 3 秒自动进入详情页
🥚 当时我长按着转了一下午
--开发者
详细数据是什么？

详细数据按表格列出分类/任务的时长与次数，支持筛选、排序和逐步展开更多行。

- 点击表头可按总时间、次数、平均时间排序（再次点击切换升/降序）。
趋势演变是什么？

趋势演变展示近 7/30 天的每日获得/消费走势，按分类或任务堆叠，直观看到哪些项目在拉升或拖累净时间。

交互方式
- 点击有数据的柱子可直接进入当日详情
- 长按柱子显示提示，可拖动切换日期
- 长按 3 秒自动进入详情页
- 快速左右滑动可切换获得/消费图表
自动检测补录

开启后，系统在每日首次启动时按设备读取关联应用时长，并与您的记录对比后补录/修正。

🤖 漏记补录：实际使用 > 已记录 → 自动补录差额（有一定惩罚）
🔧 多记修正：已记录 > 实际使用 → 自动扣减差额（有一定惩罚）
🔗 多设备：各设备记录自动汇总，避免互相覆盖
🚀 首次启用：不追溯历史，仅从启用当日/次日开始检测

提示：如果自动检测补录失败，可在任务菜单点“🤖 自动检测”触发。

📱 屏幕时间管理

屏幕时间管理帮助您控制手机使用，将节省的时间转化为奖励。

🎯 每日限额：设定每日屏幕使用时间上限
🟢 节省奖励：实际使用 < 限额 → 差额作为奖励存入余额
🔴 超出扣除：实际使用 > 限额 → 超出部分从余额扣除
⚡ 自动结算：每日首次启动时自动结算前一天
🔗 多设备：按设备独立结算，云端自动汇总
📊 分类归属：可在设置中选择奖励/扣除归属的分类

提示：点击或滑动卡片顶部可展开/收起，点击展开后的卡片可进入屏幕时间详情页。

🪟 悬浮窗计时器

悬浮窗计时器让您在使用其他应用时也能随时查看任务进度，点击悬浮窗可快速切换。

⏱️ 计时模式：普通/消费任务显示正计时，达标任务显示倒计时
📱 智能跳转：点击悬浮窗跳转到关联应用或回到时间银行
📚 多任务堆叠：存在多个悬浮窗时，自动堆叠，长按可展开/再次堆叠

提示：设置中为全局开关，任务编辑页可单独设置每个任务的悬浮窗。
