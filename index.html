版本 v7.28.0 (2026-03-29) 🛡️ 多设备同步防护 + 增量同步升级

  • [Fix] 🛡️ 彻底解决"多设备同步后数据回退"：修复长期不活跃的端（网页版或备用手机）重新联网后，将旧数据覆盖到云端导致其他设备余额回退的问题。现在任何端重连后都必须先从云端拉取最新数据，期间禁止写入，彻底杜绝旧数据污染。
  • [Perf] ⚡ Watch 重建改为增量同步：以前每次网络波动或切回前台触发 Watch 重建，都会把全部交易记录重新拉一遍（2000 条就读 2000 条）。现在 30 分钟内只拉"有变化的部分"，速度极大提升，同时节省 CloudBase API 调用次数。
  • [Feat] ☁️ 云函数幂等写入防重复：新增服务端云函数 timebankSync，新增交易时先校验云端是否已存在，存在则跳过，彻底杜绝网络重试导致的重复记录。
  • [Fix] 🔢 撤回交易后余额计算修复：修复已撤回的交易在某些情况下仍被计入余额，导致 Watch 刷新后余额莫名偏高的问题。

版本 v7.27.0 (2026-03-28) 🧹 代码架构整理

版本 v7.26.1 (2026-03-28) 📂 JS 文件智能拆分 · 编辑效率大幅提升

活动日历是什么？

活动日历以颜色强度呈现每日净时间，直观看到一段时间内的努力密度。

颜色含义
- 净增加：浅绿 0-1小时；中绿 1-3小时；深绿 3小时以上
- 净减少：浅红 0-1小时；中红 1-3小时；深红 3小时以上
交互方式
- 长按显示详情弹窗，可滑动切换日期
- 长按 3 秒后自动进入日详情页
时间仪表盘是什么？

时间仪表盘汇总你在所选周期内的时间表现。

交互方式
- 点击饼图扇区可直接进入分类/任务详情
- 长按显示详情弹窗，可拖动切换其他扇区
- 长按 3 秒自动进入详情页

趋势演变展示近 7/30 天的每日获得/消费走势，按分类或任务堆叠，直观看到哪些项目在拉升或拖累净时间。

- 长按柱子显示提示，可拖动切换日期
- 长按 3 秒自动进入详情页
- 快速左右滑动可切换获得/消费图表
自动检测补录

开启后，系统在每日首次启动时按设备读取关联应用时长，并与您的记录对比后补录/修正。

🤖 漏记补录：实际使用 > 已记录 → 自动补录差额（有一定惩罚）
🔧 多记修正：已记录 > 实际使用 → 自动扣减差额（有一定惩罚）
🔗 多设备：各设备记录自动汇总，避免互相覆盖
🚀 首次启用：不追溯历史，仅从启用当日/次日开始检测

提示：如果自动检测补录失败，可在任务菜单点“🤖 自动检测”触发。

🟢 节省奖励：实际使用 < 限额 → 差额作为奖励存入余额
