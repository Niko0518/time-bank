# Checklist

- [x] 7 个位置的版本号全部更新为 `v9.3.1`
- [x] 云函数 `startSleep` 在 `isSleeping === false` 时成功写入，返回 `code: 0`
- [x] 云函数 `startSleep` 在 `isSleeping === true` 时拒绝，返回 `code: 1002` + `startedByDeviceName`
- [x] 云函数 `endSleep` 在 `isSleeping === true` 时清空字段并返回 `code: 0`
- [x] 云函数 `endSleep` 在 `isSleeping === false` 时返回 `code: 410`（幂等）
- [x] 云函数 `endSleep` 接受 `cancelled: true` 时写入 `cancelled: true` 字段
- [x] `startUnifiedSleep` 仅在云端 `code === 0` 时设本地闹钟
- [x] `startUnifiedSleep` 在 `code === 1002` 时撤销本地闹钟 + 提示
- [x] `startUnifiedSleep` 在网络异常时撤销本地闹钟 + 提示"网络异常"
- [x] `startUnifiedSleep` 不再做"先写本地再异步同步"链式逻辑
- [x] `endUnifiedSleep` 先调云函数再执行本地结算
- [x] `endUnifiedSleep` 支持"远端结束"分支（云端 `isSleeping=true` 但本机 `false`）
- [x] `endUnifiedSleep` 远端结束使用云端 `sleepStartTime` 做结算
- [x] `cancelSleep` 走 `endSleep({ cancelled: true })` 云函数
- [x] Profile Watcher 收到 `isSleeping: true→false` 时，本机撤销闹钟 + 停监控
- [x] Profile Watcher 收到 `cancelled: true` 时通知文案为"已被取消"
- [x] `saveSleepState` 写云端时使用 `sleepState` 单一字段，不再写 `deviceSleepState.${deviceId}`
- [x] `applySleepStateFromCloud` 接收云端 `sleepState` 单一对象
- [x] `applySleepStateFromCloud` 按 `lastUpdated` 比较覆盖本地 UI 缓存
- [x] `initSleepSettings` 不再以本地 `sleepState` 为权威来源
- [x] `updateSleepCard` 显示"远端睡眠中（[设备名]）"状态
- [x] 一次性迁移函数 `migrateDeviceSleepStateToUnified` 在 `initSleepSettings` 中调用
- [x] 迁移函数取 `deviceSleepState` 中时间戳最大的活跃项写入新 `sleepState`
- [x] 迁移函数删除旧 `deviceSleepState` 字段并写回云端
- [x] 本地 `sleepState` 不再被读为权威值（仅 UI 占位）
- [x] 网络异常时 `startSleep` / `endSleep` 直接报错，无离线回退
- [x] 所有新增/修改日志包含 `[v9.3.1]` 标记
