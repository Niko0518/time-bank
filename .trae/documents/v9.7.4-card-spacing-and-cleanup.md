## 3. 变更详情（Proposed Changes）

### 3.1 Bug 1：堆叠容器残留 `st-expanded` 导致 12px 间隙

**根因**：

* 旧版 `updateStackedContainerVisibility` 在屏幕时间卡片隐藏时，仍可能保留容器的 `st-expanded: true`

* 当只有睡眠卡片可见时，CSS `body.flat-style:not(.glass-mode) .stacked-cards-container.st-expanded .screen-time-wrapper::before`（或同类规则）会注入 12px 上 margin

* 睡眠卡片因 `first-visible-card` 类抵消了自身上方 margin，但容器的 margin-top 仍作用于外层，导致余额卡片与睡眠卡片之间留出 12px 空隙

**修复**：

```javascript
// [v9.7.4] 修复：容器的 st-expanded 改由"实际可见卡片"的展开状态决定
const anyVisibleExpanded =
    (screenTimeVisible && screenTimeWrapper && screenTimeWrapper.classList.contains('expanded')) ||
    (sleepVisible       && sleepWrapper       && sleepWrapper.classList.contains('expanded'));
stackedContainer.classList.toggle('st-expanded', anyVisibleExpanded);
```

**为何不直接删除 CSS 规则**：

* 12px margin 在"屏幕时间 + 睡眠均可见"且任一展开时仍是必需的视觉间距

* 容器级 `st-expanded` 状态是共享的，只有"按可见卡片动态算"才正确

**睡眠卡片点击同步**：

* 旧代码：`handleSleepCardClick` 切换 `expanded` 类后未通知容器，导致睡眠收起时容器仍显示 `st-expanded`

* 新代码：点击后调用 `updateStackedContainerVisibility()` 重新计算（用 `typeof` 守卫防未来加载顺序变化）

### 3.2 Bug 2：设置页冗余入口

**删除理由**：

| 入口     | 删除理由                                                                                            |
| ------ | ----------------------------------------------------------------------------------------------- |
| 修复习惯连胜 | 习惯连胜自 v9.0.7\~v9.0.10 多轮修复后稳定，v9.4.0 重构后又经历 4 个版本回归测试，未见"连胜突然清零"重现                              |
| 重算余额   | 余额自 v9.1.0 起改为信任云端 `tb_profile.cachedBalance`，云端通过 `tbMutation` action=11 原子扫描维护；前端不再触发该 action |

**保留的诊断入口**：

* `DAL.recalculateBalance` 函数本体保留（`app-auth.js` L2540 注释明确说明），供未来控制台手动调用

* 数据导出/导入、`trustThisDeviceAsAuthoritative`、`showFailedMutations` 等仍保留
