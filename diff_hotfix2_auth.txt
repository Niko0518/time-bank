    v9.0.6 hotfix-2: 修复分类颜色失效（机制层：尝试修复 plain object 而非静默清空）
    
    诚实承认：v9.0.6 hotfix 把"显式抛错"改成"静默清空"，导致用户颜色"消失"。
    根因：localStorage/cloud profile 中 categoryColors / collapsedCategories 字段以 plain object
    形式存在（如 {cat1: '#fff'}），hotfix-1 直接置空导致用户颜色丢失。
    
    hotfix-2 改进：
    - setCategoryColors: 接收 plain object 时用 Object.entries 转 [[k, v]]
    - setCollapsedCategories: 接收 plain object 时用 Object.keys 转 [k]
    - getLocalData: 同样尝试修复，只在空 plain object 时才清空
    - initApp 早期自愈: 同样尝试修复，只在空 plain object 时才清空
    - 只在"无法修复"（如 {}/null/Date）时降级为空，并打 console.warn
    
    逻辑：
      - 正常数组 → 通过
      - plain object {cat1: color1, ...} → Object.entries() 修复为 [['cat1', color1], ...]
      - 空 plain object {} / null / undefined → 降级为空（且备份到 *Corrupted 键）
      - 其他奇怪类型（Date 等）→ 降级为空
    
    版本号不变（用户确认保持 v9.0.6）
    
    Co-Authored-By: Claude <noreply@anthropic.com>
