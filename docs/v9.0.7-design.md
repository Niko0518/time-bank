# v9.0.7 设计文档：习惯系统重构

> **版本**：v9.0.7
> **日期**：2026-06-04
> **作者**：AI 编程助手
> **状态**：✅ 已实施
> **关联**：v9.0.5 修复后暴露的"连胜突然清零"bug

---

## 1. 背景

v9.0.5 修复了"任务复活数据损坏"和"onRollback 完整化"问题后，6.3 用户反馈：

| 任务 | stored streak | computed streak（重算） | 说明 |
|------|---------------|-------------------------|------|
| 吃早饭 | 6 | 6 | ✅ 正常 |
| 晨起洗漱 | 1 | 13 | ❌ 14 天连胜变 1 |
| 维生素/叶黄素片 | 1 | 1 | ❌ 6.3 完成 2 笔无奖励 |
| 夜间洗漱 | 1 | 15 | ❌ 16 天连胜变 1 |

用户要求"v9.0.1 之后什么改动导致的问题，如何修复"。

经排查，**根因在 v7.37.0 引入 `transactionIndex` 时埋下，v9.0.0/v9.0.1 之后才暴露**。

---

## 2. 根因分析

### 2.1 触发链

```
[v7.37.0] 引入 transactionIndex（Map<taskId, Transaction[]>）
           作用：让 rebuildHabitStreak 用 O(1) 替代 O(n) filter
           副作用：rebuildHabitStreak 优先用 transactionIndex.has(task.id)

[v9.0.0] 服务端权威写入架构
           影响：applyDataState 路径下 transactionIndex 永远不会被构建
                （之前架构下 initApp 会从云端 loadAll 触发 buildTransactionIndex）

[v9.0.2] onRollback 引入
           副作用：addTransaction.onRollback 只清理 transactions 数组
                   不清理 transactionIndex → 索引残留

[6.3 用户场景]
           1. applyDataState 加载本地缓存（索引为空）
           2. 用户点完成"夜间洗漱"
           3. addTransaction 同步把交易加到 transactions 数组和 transactionIndex
           4. rebuildHabitStreak：transactionIndex.has(task.id)=true
              走索引路径，但索引中只有这 1 笔交易
           5. streak = 1
           6. task.habitDetails.streak = 1 覆盖原来的 15
           7. DAL.saveTask 同步到云端 → 持久化 streak=1
```

### 2.2 3 个 P0 子 bug

| 编号 | Bug | 影响 |
|------|-----|------|
| P0-1 | `applyDataState` 后 `transactionIndex` 为空，`rebuildHabitStreak` 走索引路径算出 streak=1 | 所有走本地缓存的完成操作都会"清零连胜" |
| P0-2 | `onRollback` 不清理 `transactionIndex` | 失败重试场景下索引残留，数据漂移 |
| P1-1 | `processHabitCompletion` 两次读 `task.habitDetails.streak` 漂移 | 奖励发放与连胜状态可能不一致 |

---

## 3. 修复方案

### 3.1 最小修复（1 行）

```javascript
// app-auth.js: applyDataState 末尾
if (typeof buildTransactionIndex === 'function') {
    buildTransactionIndex();
}
```

**优点**：最小风险，1 行代码
**缺点**：
- 不解决 P0-2（onRollback 索引残留）
- 不解决 P1-1（两次读漂移）
- 不提供"修复历史数据"路径

#### 改动 5：rebuildHabitStreak 单一数据源

**位置**：[app-2.js:6014](file:///d:/TimeBank/android_project/app/src/main/assets/www/js/app-2.js#L6014)

```javascript
// 之前
const taskTxs = (typeof transactionIndex !== 'undefined' && transactionIndex.has(task.id))
    ? transactionIndex.get(task.id)
    : transactions.filter(t => t.taskId === task.id);

// 之后
const taskTransactions = transactions
    .filter(t => t.taskId === task.id && t.type === 'earn' && !t.undone)
```

## 4. 风险评估

| 改动 | 风险 | 缓解措施 |
|------|------|---------|
| applyDataState 加 buildTransactionIndex | 低 | 纯加法，不影响其他路径 |
| onRollback 加索引清理 | 中 | 仅 onRollback 路径触发，正常路径不变 |
| rebuildHabitStreak 改用 transactions.filter | 中 | 性能：之前 O(1) 现在 O(n)——但 n 通常 < 100，可接受 |
| rebuildHabitStreak 返回值 | 低 | 不破坏现有调用方兼容性 |
| processHabitCompletion 用返回值 | 低 | 消除 2 次读漂移，行为更一致 |
| fixAllHabitStreaks + UI | 低 | 仅手动触发，不影响自动流程 |

---

## 5. 兼容性

- **不破坏现有数据**：所有改动都是"读"路径的修复，不修改历史数据
- **可手动修复历史**：用户提供 UI 按钮触发修复
- **云端无需部署**：纯客户端修复
